GitHub (for SEO auto-commit)

When SEO sends you a Telegram card and you tap ✓ Apply, the change is committed straight to your repo's main branch. Token needs Contents: Read & Write on the target repo only.

Create at github.com/settings/tokens?type=beta → "Generate new token" → only your repo → Contents: Read & Write.

Connect your own API keys below. All keys are encrypted at rest (AES-256-GCM) and never exposed to the browser. Click "Test" to verify a connection works.

Quick setup guide
  1. Claude: Get your API key from console.anthropic.com and paste it when you click Connect.
  2. Telegram: Message @BotFather, create a bot, copy the token. Send /start to your bot, then visit https://api.telegram.org/bot<TOKEN>/getUpdates and copy chat.id. Click Connect and paste both.
  3. Twitter/X: At developer.x.com, create a Web App. Add callback URL https://yourdomain.com/api/pa/oauth/twitter-callback. Enable scopes tweet.read tweet.write users.read media.write offline.access. Click Connect, paste Client ID + Secret, authorize.
  4. LinkedIn: At developer.linkedin.com/apps, create an app. Under Products, request "Share on LinkedIn" and "Sign In with LinkedIn using OpenID Connect". Add redirect URL https://yourdomain.com/api/pa/oauth/linkedin-callback. Click Connect, paste Client ID + Secret, authorize.
  5. Test everything: Click "Test" on each service to verify.
  6. Run Writer: Go to the Agents tab and click "Run now" on Writer to generate your first draft — it'll auto-render a branded image card when appropriate.
  7. Publish: Once Writer → Editor → Messenger flow the draft to Telegram, tap ✓ Approve, then Run Publisher — it posts to Twitter AND LinkedIn with the same branded PNG.
